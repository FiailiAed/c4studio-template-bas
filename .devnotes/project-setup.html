3. TailwindCSS v4 + Base Layout

Install TailwindCSS v4 via the Vite plugin (no config file needed). Create global CSS, base layout, and path aliases.

Install

bun add tailwindcss @tailwindcss/vite

astro.config.mjs — add Vite plugin

import tailwindcss from '@tailwindcss/vite';

export default defineConfig({ vite: { plugins: [tailwindcss()] } });

src/styles/global.css

@import "tailwindcss";

tsconfig.json — path aliases (baseUrl required)

"baseUrl": ".",

"paths": { "@pages/*": ["src/pages/*"], "@layouts/*": ["src/layouts/*"], "@styles/*": ["src/styles/*"] }

NOTE: Astro reads tsconfig paths natively — no vite.resolve.alias needed. Always add a new alias here when creating a new src/ subfolder.

src/layouts/BaseLayout.astro

Imports @styles/global.css and astro:transitions ClientRouter.

Props: title (required), description, image, schema (JSON-LD).

Named slots: head, header, footer. Default slot renders inside <main id="main">.

Includes skip-to-content link, Open Graph tags, Twitter Card tags, and optional canonical URL (set astro.site in config to enable).

src/pages/index.astro — use the layout

import BaseLayout from '@layouts/BaseLayout.astro';

Wrap all page content in <BaseLayout title="..."> ... </BaseLayout>

4. Vercel Web Analytics

Install @vercel/analytics and add the Astro component to BaseLayout so every page is tracked automatically.

Install

bun i @vercel/analytics

src/layouts/BaseLayout.astro — import (default, not named)

import Analytics from '@vercel/analytics/astro';

NOTE: The package exports a default from dist/astro/index.astro. Use a default import — import { Analytics } will resolve to undefined and throw a runtime error.

src/layouts/BaseLayout.astro — place component

Add <Analytics /> just before </body>, after the footer slot.
