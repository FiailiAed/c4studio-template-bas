Middleware gotcha

In @clerk/astro v3, the middleware handler receives auth as a FUNCTION. Call auth() to get the auth object with redirectToSignIn(). auth.protect() does NOT exist — calling it throws "auth.protect is not a function".

Correct pattern: const authObject = auth(); if (!authObject.userId) return authObject.redirectToSignIn();

Three user-created entity types with identical data shape. Each gets a Convex table, a list page, and a per-instance edit page. The admin portal index fetches all three at render time and shows one card per instance plus a "Create new" dashed card at the end of each section.

convex/schema.ts — three new tables

funnels — name (string), slug (string), description (optional string), published (boolean), headline (optional string), subheadline (optional string), ctaLabel (optional string), ctaHref (optional string).

shops — name, slug, description, published, headline, subheadline. Products managed via shopItems child table (shopId FK, name, description, price in cents, currency, stripePriceId, imageUrl, published, order). Each exports: list (query, ordered desc, take 100), get (query by id), getBySlug (query by slug index), create (mutation), update (mutation — patches by id), remove (mutation).

getBySlug added to funnels.ts, shops.ts, and bookingLinks.ts. New modules: convex/bookings.ts (CRUD + listByDate), convex/shopItems.ts (CRUD by shopId). Stack now includes @astrojs/svelte for the booking widget island.

Upload flow — fully server-side (no browser JS required for upload)

1. Form submits as multipart/form-data with action=upload to the Astro SSR POST handler.

2. Server reads the File from formData.get('file') and determines mediaType from file.type (image/* → image, video/* → video, application/pdf → pdf).

3. Server calls client.mutation(api.gallery.generateUploadUrl, {}) to get a short-lived Convex upload URL.

4. Server POSTs the file to that URL: fetch(uploadUrl, { method: 'POST', headers: { 'Content-Type': mimeType }, body: file }).

5. Convex returns { storageId }. Server calls api.gallery.create with storageId + metadata.

NOTE: Client-side drag-and-drop enhancement is present (updates the filename label and injects dropped files into the input), but the actual upload always goes through the SSR POST handler.

Design decision: no Astro <Image> for Convex media

Maintenance overlay UI

When maintenanceActive is true, renders a fixed inset-0 z-[99999] div over the entire page. This approach avoids complex conditional slot rendering — the page slots still render behind the overlay but are visually hidden.

Design: bg-slate-900 full-screen, centered flex column. Wrench SVG icon in a rounded container, heading "We'll be right back." (Lexend), subheading "We're making some improvements. Check back soon." (Inter, slate-400), appName shown in small slate-600 text below.

Go to /admin/settings → Feature Flags. Toggle Maintenance Mode on. All public pages immediately show the overlay. Admin portal remains fully accessible. Toggle off to restore the site.
